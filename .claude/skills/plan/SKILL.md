# Feature Planning Skill
1. Using the /spec-v2 command, create a feature spec in _specs/<feature-name>.md
2. Create an implementation plan from the new spec in _plans/<feature-name>.md with numbered phases
3. Create a feature branch: git switch -c feature/<feature-name>
4. Using the /commit-message-v2 command, commit the spec and plan docs
5. Summarize what was created and ask if user wants to start implementation
